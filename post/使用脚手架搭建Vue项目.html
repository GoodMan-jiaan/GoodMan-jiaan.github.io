使用脚手架搭建Vue项目

发表于2021-03-25|更新于2024-09-17|前端框架
|字数总计:568|阅读时长:2分钟|阅读量:

Vue.js安装

  • 直接CDN引入

    你可以选择引入开发环境版本还是生产环境版本

  • 下载和引入

    开发环境 https://vuejs.org/js/vue.js

    生产环境 https://vuejs.org/js/vue.min.js

  • npm安装

    通过webpack和CLI的使用，我们使用该方式。

vue-cli

介绍

Vue CLI 是一个基于 Vue.js 进行快速开发的完整系统，提供：

  • 通过 @vue/cli 实现的交互式的项目脚手架。
  • 通过 @vue/cli + @vue/cli-service-global 实现的零配置原型开发。
  • /cli-service)，该依赖：
    • 可升级；一个运行时依赖 (@vue
    • 基于 webpack 构建，并带有合理的默认配置；
    • 可以通过项目内的配置文件进行配置；
    • 可以通过插件进行扩展。
  • 个丰富的官方插件集合，集成了前端生态中最好的工具。
  • 一套完全图形化的创建和管理 Vue.js 项目的用户界面。

安装

  • 如果安装过vue-cli (1.x 或 2.x)，可以先进行卸载
  • 安装最新版本

创建项目

  1. 目录来到想建立的目录下，打开控制台

  2. 创建一个项目

    • 选择手动配置

    • 根据自己需求选中一下选项(空格为选中，回车为确定)

    • 选择vue版本（我一般使用的是vue3）

    • 选择class-style、babel、路由模式、css预处理(根据自己的需要选择)

    • eslint校验以及校验时机（我选择提交时校验）

    • 选择 Babel, PostCSS, ESLint 等自定义配置的存放位置（我选择在各自配置文件，vue3上默认时放在package.json文件里）

    • 是否将以上配置 作为将来项目的预置吗？
