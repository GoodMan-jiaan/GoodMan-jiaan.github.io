JDBC连接数据库

发表于2025-05-16|更新于2025-05-17|开发基础
|字数总计:1.4k|阅读时长:4分钟|阅读量:

PS: 有了mybatis、jpa等技术框架，为什么我还要重过一遍JDBC呢？

  1. 理解底层原理
    • ORM 框架的本质，MyBatis 和 JPA（如 Hibernate）等框架本质上是对 JDBC 的封装。学习 JDBC 能帮你理解这些框架如何管理数据库连接、执行 SQL、处理事务等底层机制。
    • 调试和问题排查，当框架生成的 SQL 出现性能问题或错误时，了解 JDBC 可以让你直接分析底层执行的 SQL、参数绑定、结果集处理等，快速定位问题根源。
  2. 灵活应对复杂场景
    • 复杂 SQL 或存储过程，在需要高度优化或复杂 SQL 的场景（如多表关联、动态条件查询），直接使用 JDBC 可以更灵活地控制 SQL 编写和执行逻辑，避免 ORM 框架的局限性。
    • 批量操作和性能优化，JDBC 提供原生的批量处理（addBatch()、executeBatch()）和手动事务控制能力，允许你在需要极致性能的场景下绕过 ORM 框架的抽象层。
  3. 掌握数据库交互的核心流程
    • 连接管理，学习 JDBC 能让你理解数据库连接的获取、释放、连接池（如 HikariCP）的工作原理，避免连接泄漏或资源浪费。
    • 事务控制，通过 JDBC 手动控制事务（setAutoCommit(false)、commit()、rollback()），深入理解事务的隔离级别、传播行为等概念。
    • 结果集处理，学习 ResultSet 的遍历、类型转换和资源释放，理解 ORM 框架是如何封装这些操作的。
  4. 解决框架的局限性
    • 框架无法覆盖的场景，某些数据库专有语法或函数（如 Oracle 的 CONNECT BY）。动态 SQL 拼接（MyBatis 虽支持，但复杂逻辑仍需手动优化）。需要直接操作 JDBC 驱动的高级功能（如 PostgreSQL 的 COPY 命令）。
    • 性能敏感场景，ORM 框架的自动映射和反射机制可能带来性能损耗。通过 JDBC 手动优化可以绕过这些开销。
  5. 维护遗留代码或特殊项目
    • 老旧系统维护，许多遗留项目仍直接使用 JDBC，熟悉 JDBC 能帮助你快速接手和维护这类代码。
    • 轻量级需求，在小型项目或工具类代码中，直接使用 JDBC 可能比引入 ORM 框架更简洁高效。
  6. 技术面试的必备知识

初识JDBC

JDBC是什么？JDBC英文名为：Java Data Base Connectivity(Java数据库连接)，官方解释它是Java编程语言和广泛的数据库之间独立于数据库的连接标准的Java API，根本上说JDBC是一种规范，它提供的接口，一套完整的，允许便捷式访问底层数据库。可以用JAVA来写不同类型的可执行文件：JAVA应用程序、JAVA Applets、Java Servlet、JSP等，不同的可执行文件都能通过JDBC访问数据库，又兼备存储的优势。简单说它就是Java与数据库的连接的桥梁或者插件，用Java代码就能操作数据库的增删改查、存储过程、事务等。

我们可以发现，JDK自带了一个java.sql包，而这里面就定义了大量的接口，不同类型的数据库，都可以通过实现此接口，编写适用于自己数据库的实现类。而不同的数据库厂商实现的这套标准，我们称为数据库驱动。
